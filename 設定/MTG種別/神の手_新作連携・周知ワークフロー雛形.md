# 新作（新メニュー）プロジェクト 実行ガイド — 岡さん用

**岡さんがプロジェクトリーダー**として、新作1本を「企画→提供開始→告知」まで進めるときの実行用雛形です。あなたが誰に何を依頼し、いつ何を確認すればよいかをまとめています。

---

## 1. 岡さんの役割と、まわりの担当

**岡さんの役割**：新作プロジェクトの**とりまとめ・スケジュール管理・原さんとの連携**。提供開始日を軸に、後藤さん・原さん・吉浦・柿本AI・ポテに「いつまでに何を出すか」を伝え、抜けなく進める。

| チーム | 担当 | 新作でやること（岡さんが依頼・確認する相手） |
|--------|------|----------------------------------------------|
| **現場（店舗）** | 後藤さん | 試作・レシピ原案・動画 or 写真（柿本AI用）。あなた→「〇日までにレシピ原案ください」と依頼。 |
| | 原さん | 店舗での試作・品質、**スタッフへの周知**。あなた→開始日・レシピ・ルールを原さんに渡し、店舗内で共有してもらう。 |
| **運営** | 吉浦 | レシピ最終調整・原価・メニュー表・チラシ・メディア用文言・画像。あなた→締切を伝え、上がってきたものを原さん・ポテに回す。 |
| | 柿本AI | 動画→レシピ文。あなた→後藤さんの動画を渡し、たたき台レシピをもらう（吉浦へつなぐ）。 |
| **メディア** | ポテ | 提供開始日前にSNS・LINE等で告知。あなた→「開始日・メニュー名・画像は吉浦から」と伝え、告知日をポテと決める。 |

**原則**：全員が同じ**提供開始日**と**メニュー名**で動く。開始日はあなたが原さん・吉浦・ポテとすり合わせて決め、一度決めたら全チームに共有する。

---

## 2. 全体の流れ（岡さんが押さえる順番）

1. 企画・コンセプト（あなたがとりまとめ）  
2. 試作・レシピ原案（後藤さん・原さん）  
3. 動画→レシピ（柿本AI）→ レシピ・原価確定（吉浦）  
4. メニュー表・チラシ・メディア用素材（吉浦）  
5. **提供開始日を確定**（あなたが原・吉浦・ポテと決定）  
6. 店舗へ周知（あなた→原さんに渡す。原さん→スタッフへ）  
7. メディア告知（ポテ。あなたが告知日を確認）  
8. 提供開始  

---

## 3. ステップごと：岡さんがやること・依頼すること

| ステップ | 岡さんがやること | 誰に依頼するか・何を確認するか |
|----------|------------------|--------------------------------|
| **① 企画・コンセプト** | 新作の「何を・誰向け・いつ頃」をとりまとめる。後藤・原・吉浦と話して方向を決め、**希望の提供時期**を出してもらう。 | 後藤・原・吉浦に「〇月頃に出したい」「こんなコンセプト」を共有し、実現可能か聞く。 |
| **② 試作・レシピ原案** | 後藤さんに**締切**を伝え、レシピ原案と動画（or 写真）をもらう。原さんが店舗で試作・味を見ているか確認。 | 後藤さん「〇日までにレシピ原案と動画（柿本AI用）ください」。原さん「試作の進捗どうですか」。 |
| **③ 動画→レシピ** | 後藤さんの動画を柿本AIに渡し、たたき台レシピをもらう。吉浦に「たたき台できたので最終調整お願いします」とつなぐ。 | 柿本AIに動画を渡す。吉浦にたたき台を渡し、締切を伝える。 |
| **④ レシピ・原価確定** | 吉浦に**締切**を伝え、確定レシピ・原価率をもらう。後藤・原に「これで確定でいいか」確認してもらう。 | 吉浦「〇日までに確定レシピ・原価ください」。確定後、後藤・原に共有。 |
| **⑤ メニュー表・チラシ・メディア用** | 吉浦にメニュー表・チラシ・SNS用の文言・画像の**締切**を伝える。できたら原さん・ポテに渡す。 | 吉浦「〇日までにメニュー表・チラシ・告知用文言・画像ください」。ポテに「告知はこの日、素材は吉浦から」と伝える。 |
| **⑥ 提供開始日・店舗周知** | **あなたが提供開始日を決める**（原・吉浦・ポテと調整）。開始日・終了日（あれば）・レシピ・ルールを**原さんに渡す**。原さんが店舗スタッフに共有する。 | 原さん「〇日提供開始で、レシピ・ルールはこれです。店舗のみんなに共有お願いします」。食材・器具の手配は後藤・原と確認。 |
| **⑦ メディア告知** | ポテに**告知日**を伝え、「開始日・メニュー名・画像は吉浦から受け取った」と確認。告知前に原さんに「店舗の準備OKか」聞く。 | ポテ「〇日に告知お願いします。文言・画像は吉浦から」。原さん「〇日提供開始で店舗準備大丈夫ですか」。 |
| **⑧ 提供開始後** | 売れ行き・クレーム・SNSの反応を、原さん・ポテから聞き、次の新作やルールに反映するか判断。 | 原さん・ポテに「反応どうでしたか」と定期的に聞く。 |

## 4. 岡さんのスケジュール目安（提供開始日を D 日とする）

| 時期 | 岡さんがやること |
|------|------------------|
| **D-3週間〜** | 企画・コンセプトを固め、後藤・原に試作を依頼。**「〇月〇日提供開始」の案**を出し、実現可能か確認。 |
| **D-2週間** | 柿本AI→吉浦でレシピ確定。吉浦にメニュー表・チラシ・告知用の締切を伝える。**提供開始日を正式に決め**、原・吉浦・ポテに共有。 |
| **D-1〜2週間** | **開始日・レシピ・ルールを原さんに渡し、店舗周知をお願いする。** ポテに告知日と「素材は吉浦から」を伝える。 |
| **D-1週間** | ポテが告知する。原さんに「店舗の準備・食材・練習はOKか」最終確認。 |
| **D 日** | 提供開始。当日は原さん・後藤さんに店舗の様子を確認。 |
| **D 以降** | 原さん・ポテから反応を聞き、次に活かす。 |

※ 期間限定・数量限定の場合は、**終了日・ルール**もこのタイミングで全員に共有する。

---

## 5. 岡さんが確認するチェックリスト（新作1本ごと）

提供開始前に、あなたが「伝えたか・揃ったか」を確認するリスト。

- [ ] **提供開始日・終了日（あれば）**を、原さん・吉浦・ポテに伝えたか
- [ ] **原さん**に、開始日・レシピ・盛り付け・接客のポイント・終了日を渡し、店舗スタッフへ共有してもらったか
- [ ] **吉浦**から、確定レシピ・メニュー表・チラシ・告知用文言・画像を受け取り、原さん・ポテに渡したか
- [ ] **ポテ**に告知日を伝え、吉浦から素材を受け取ったか・告知前に原さんと店舗準備を確認したか
- [ ] **提供開始後**、原さん・ポテから売れ行き・反応を聞き、メモしたか（次回の新作に活かす）

---

## 6. この雛形の使い方（岡さん向け）

- **新作を1本決めたら**：このファイルを開き、**提供開始日（D）**を決める。上記のタイムラインで「D-2週間で何をするか」を逆算する。
- **毎週 or MTGで**：「今、どのステップか」「誰に何を依頼したか」「締切は守られそうか」を確認する。チェックリストで抜けがないか見る。
- **締切・担当が不明なとき**：社長に「〇の締切いつにしますか」「〇は誰に依頼しますか」と聞く。
