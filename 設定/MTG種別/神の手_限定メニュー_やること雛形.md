# 限定メニュー（期間限定・数量限定）会議 — やること雛形

**用途**：限定メニューの会議で決まったことを、議事録・Todo に落とすときのフォーマット。新作と同様の流れに加え、**提供期間・終了日・数量ルール**を必ず明記する。

---

## 1. 決まったこと（議事録冒頭で書く項目）

| 項目 | 内容（例） |
|------|------------|
| **メニュー名** | 〇〇ラーメン 期間限定 など |
| **提供開始日** | 〇月〇日 |
| **提供終了日** | 〇月〇日（または「〇週間」「在庫なくなり次第」など） |
| **限定ルール** | 1日〇食限定 / 先着〇名 / 〇店舗のみ など（あれば） |
| **価格** | 〇〇円（税込） |
| **告知日** | SNS・LINE等で〇月〇日に告知 など |

---

## 2. 誰が何をすべきか（担当｜やること｜期限）

以下の表を埋める。期限が未定の場合は「未定」「次回MTGで確定」と書く。

| 担当 | やること | 期限 |
|------|----------|------|
| **岡** | 提供開始日・終了日を原・吉浦・ポテと確定し、全員に共有する | 〇月〇日 |
| **岡** | 開始日・終了日・レシピ・限定ルールを原さんに渡し、店舗スタッフへ共有してもらう | 提供開始の〇日前 |
| **後藤・原** | 試作・レシピ原案・動画（柿本AI用）（新作と同様） | 〇月〇日 |
| **吉浦** | 確定レシピ・メニュー表・チラシ・告知用文言・画像（終了日・限定ルールを明記） | 〇月〇日 |
| **ポテ** | 提供開始前にSNS・LINEで告知（開始日・終了日・限定ルールを記載） | 〇月〇日 |
| **原** | 店舗スタッフへ開始日・終了日・レシピ・盛り付け・接客のポイント・限定ルールを共有 | 提供開始の〇日前 |

※ 新作1本と同時に進める場合は、`神の手_新作連携・周知ワークフロー雛形.md` の流れに「終了日・限定ルール」を追加して運用する。

---

## 3. 限定メニュー会議で必ず確認すること（チェックリスト）

- [ ] **提供終了日**（または終了条件）を全員で共有したか
- [ ] **限定ルール**（1日〇食、先着〇名など）を原さん・吉浦・ポテに伝えたか
- [ ] 原さん経由で**店舗スタッフに終了日・限定ルール**を周知する担当と日付を決めたか
- [ ] 告知文に**終了日・限定ルール**を入れるようポテ・吉浦に依頼したか
- [ ] 終了日以降の在庫・廃棄ルールを後藤・原と確認したか（必要なら）

---

## 4. 議事録・Todo に書くときの定型

- **決まったこと**：「〇〇（メニュー名）を〇月〇日〜〇月〇日で提供。〇〇（限定ルール）。告知は〇月〇日。」
- **誰が何を**：上記「2. 誰が何をすべきか」の表をそのまま、または要約して議事録・Slack用Todoに貼る。

不明な点（締切・担当者など）は推測せず、社長に「〇の締切いつにしますか」「〇は誰に依頼しますか」と聞く。
