# ルール変更の会議 — やること雛形

**用途**：接客・調理・衛生・勤務など、ルール変更の会議で決まったことを、議事録・Todo に落とすときのフォーマット。**誰が・いつ・どの方法で店舗に伝えるか**を必ず明記する。

---

## 1. 決まったこと（議事録冒頭で書く項目）

| 項目 | 内容（例） |
|------|------------|
| **変更するルール** | 接客ルール / 調理手順 / 衛生管理 / 勤務・シフト など |
| **変更内容の要約** | 〇〇を〇〇に変更する（1行で） |
| **運用開始日** | 〇月〇日から適用 |
| **店舗への周知方法** | 原さん経由で口頭＋メモ / 原さんが朝礼で共有 / 〇〇ツールで共有 など |
| **周知完了の目安** | 〇月〇日までに全スタッフに伝える など |

---

## 2. 誰が何をすべきか（担当｜やること｜期限）

店舗周知は**原さん経由**が原則。岡が運営側で内容を整理し、原さんに渡す。

| 担当 | やること | 期限 |
|------|----------|------|
| **岡**（または決定者） | 変更内容を文書化（または箇条書き）し、原さんに渡す | 〇月〇日 |
| **原さん** | 変更内容を確認し、不明点があれば岡に確認する | 受け取り後〇日以内 |
| **原さん** | 店舗スタッフへ、〇〇の方法で周知する（朝礼・メモ・共有ツール等） | 〇月〇日まで |
| **原さん** | 運用開始日にルールが守られているか現場で確認する | 運用開始日〜 |

※ 運営チーム向けのルール変更の場合は、岡が吉浦・ポテ等に直接共有。その場合も「誰に・いつ・どの方法で」を書く。

---

## 3. ルール変更会議で必ず確認すること（チェックリスト）

- [ ] **変更内容**を1文で要約し、議事録に書いたか
- [ ] **運用開始日**を決めたか
- [ ] **店舗に伝える人**（原則：原さん）を決めたか
- [ ] **周知の方法**（口頭・メモ・ツール・朝礼等）を決めたか
- [ ] **いつまでに周知するか**（運用開始の〇日前など）を決めたか
- [ ] 原さんが「受け取った・共有した」を確認できるようにしたか

---

## 4. 議事録・Todo に書くときの定型

- **決まったこと**：「〇〇（ルール名）を〇〇に変更。〇月〇日運用開始。店舗へは原さんが〇〇の方法で〇月〇日までに周知。」
- **誰が何を**：上記「2. 誰が何をすべきか」の表をそのまま、または要約して議事録・Slack用Todoに貼る。

不明な点（運用開始日・周知方法・担当者など）は推測せず、社長に「〇はいつからにしますか」「店舗への伝達は原さんでよいですか」と聞く。
