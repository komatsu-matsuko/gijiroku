# 議事録ワークスペース 保守・更新ガイド

**誰が担当しても、同じように「議事録の頭脳」として使えるようにするための一覧です。**  
社員の増減・役職変更・制度変更・チャンネル変更などがあったとき、どのファイルをいつ更新すればよいかをまとめています。

---

## 1. 社員が増えた・減った・役職が変わったとき

| 更新する場所 | ファイル／フォルダ | やること |
|--------------|-------------------|----------|
| **スタッフ一覧** | `楽しい仲間達/` 内の一覧ファイル | **必須。** 名前・役職・所属を追加／削除／修正する。ファイル名は任意（例：`メンバー一覧.md` や `スタッフ一覧.md`）。ここにいる人物を「弊社スタッフ」として議事録・文字起こしの名前照合に使う。 |
| **記録・Todo用メンバー** | `参照情報/メンバー一覧.md` | 記録係・Todo係が参照する。**幹部・役員**の名前・役職・担当を `## 幹部` `## 役員` の下に箇条書き。楽しい仲間達と役割が違う場合は、ここを別ファイルで持ってもよい（楽しい仲間達の一覧をコピーして役職別に整理しても可）。 |
| **個人の配慮・アドバイス** | `ナレッジ/スタッフ_特性・アドバイス.md` | 入退社や役職変更に合わせ、該当者のメモを追加・削除・更新する。議事録作成・FB・所見で参照される。 |

**ポイント**

- **楽しい仲間達**：文字起こしで出てくる名前が「この一覧にいる人」かどうかの基準。いない名前は文字起こしの誤りとして修正する運用なので、**入社したら必ず追加**する。
- **参照情報/メンバー一覧.md**：議事録の記録係・Todo係が「誰に何を依頼するか」を書くときに使う。幹部・役員中心でよい場合は、楽しい仲間達の一覧と兼用でも可。

---

## 2. その都度・定期で更新が必要なファイル

| きっかけ | ファイル／フォルダ | やること |
|----------|-------------------|----------|
| **毎回のMTG後** | `設定/次回MTG.md` | 次回幹部MTGの予定日を更新。会議前ブリーフィング・リマインダーがこの日付を参照する。 |
| **Slackチャンネル・DMが変わった** | `設定/slack.md` | リマインダー送信先（チャンネルID）・議事録・FB送信先（社長のSlack ID）を書き換える。 |
| **評価制度・FBルールが変わった** | `参照情報/役員_評価制度.md` | 役員FBの6項目・配点・観点を更新。月1回の評価・FBに使う。 |
| **一般社員の評価が変わった** | `参照情報/一般社員_評価制度.md` | 課長以下がMTGに参加する場合のFB・次のタスク用。6項目を更新。 |
| **FBの書き方・運用だけ変わった** | `参照情報/評価制度・FBルール.md` | フォーマット例・運用ルールを更新（項目そのものは `役員_評価制度.md`）。 |
| **会議の型・用語が変わった** | `参照情報/会議の型・用語.md` | アジェンダのテンプレ、社内略語・プロジェクト名を更新。記録係が参照。 |
| **経営方針・アドバイザー観点が変わった** | `参照情報/経営の前提・アドバイザー観点.md` | 経営アドバイザー所見の観点を更新。 |
| **社長の人事評価指標が変わった** | `参照情報/社長用人事評価指標.md` | 社長の振り返り用（6項目・5段階）。必要に応じて更新。 |

---

## 3. 参照情報の「ファイル名」について

議事録作成時、AIは **次のファイル名** で `参照情報/` を自動参照します。名前を変えると参照されなくなるので、新規で作るときは下表の名前で作成してください。

| ファイル名 | 主な用途 |
|------------|----------|
| `メンバー一覧.md` | 記録係・Todo係（担当者・役職） |
| `役員_評価制度.md` | 役員FBの評価観点 |
| `一般社員_評価制度.md` | 一般社員がMTGに参加するときのFB・タスク |
| `評価制度・FBルール.md` | FBのフォーマット・運用 |
| `会議の型・用語.md` | 議事録の用語・アジェンダの型 |
| `経営の前提・アドバイザー観点.md` | 経営アドバイザー所見の観点 |
| `社長用人事評価指標.md` | 社長の人事評価・振り返り |

くわしくは `参照情報/README.md` と `★MTGフロー・ルール.md` の「ファイルを渡すと、より高度にする方法」を参照。

---

## 4. 更新後の確認

- 参照情報を更新したあと、「次回の議事録作成から、参照情報の最新版を使って」と AI に一度伝えておくと、以後その内容が優先される。
- 楽しい仲間達やメンバー一覧を更新したら、次回の「MTG終了」または「議事録を作成」から、新しい名前・役職が議事録・Todoに反映される。

---

## 5. このガイドの置き場所

- **このファイル**：`議事録_保守・更新ガイド.md`（議事録フォルダの直下）
- **MTGの流れ全体**：`★MTGフロー・ルール.md`
- **設定の変更手順**：`設定/README.md`

社員の増減や制度変更があった担当者は、まずこのガイドの「1. 社員が増えた・減った〜」「2. その都度・定期で〜」を確認し、該当ファイルを更新してください。
