# 音声議事録 → 議事録・Todo・役員FB ワークフロー

**★ 短い手順・役割分担・AIを賢くする情報 → [★MTGフロー・ルール.md](★MTGフロー・ルール.md)**

このPCを会議に持参し、**MTG開始 → 必要な情報の聞き返し → 前提条件の確認 → y → 音声・追加情報入力 → MTG終了で議事録作成** の流れで議事録・参加者への指示・役員FBまでまとめて出力するための手順です。

---

## 「MTG開始」からの標準フロー

1. チャットで **「MTG開始」** と送る → AIが議事録作成に**必要な情報を聞き返す**（参照情報の有無、日付・会議名・出席者など）。
2. **それに答える** → AIが**前提条件をまとめ**、「MTGを開始しますか？確認」と返す。
3. **「y」** と送る → AIが「音声・追加情報を入力し、不足情報の補足や内容の整理を一緒に行えます。準備ができたら「MTG終了」と送ってください。」と案内。
4. 音声（または文字起こし）と追加情報を入力 → **MTG終了まで**、不足情報の補足・内容の整理をAIと一緒に行う（議事録はまだ作らない）。
5. **「MTG終了」** と送る → AIが**議事録・Todo等の作成を開始**。以下と同じ成果物を出力。
6. **内容を確認し、手動でSlackに送る**（社長用送信先は 設定/slack.md の D01KDC0NN7M）。

### MTG終了（または「議事録を作成」）で出るもの

| # | 内容 |
|---|------|
| ① | 議事録＋会議参加者への指示（Todo）… メンバーのSlackに送れる文面 |
| ② | 意思決定ログ（誰に・何を・いつまでに） |
| ③ | 次回の会議前ブリーフィング |
| ④ | 役員へのFB内容（評価制度・ルールに基づく） |
| ⑤ | 経営アドバイザー所見（幹部の未熟な箇所・改善点の指摘） |
| ⑥ | 社長のSlack送信用（上記の要約。D01KDC0NN7M 用） |
| （該当時） | 「以下の点が不明瞭です。お答えください。」… 確認したいポイントの箇条書き |

チームの役割定義は **`チーム役割.md`** を参照。スキルは **`~/.cursor/skills/MTG議事録/`** にあります。

## 追加取り組み（チャットで指示するだけ）

| やりたいこと | チャットで送る文言 |
|--------------|---------------------|
| **会議前ブリーフィング** | 「会議前ブリーフィング」→ 前回のやること・進捗を整理してMTG前に共有用の文面を出力 |
| **意思決定ログ** | 「意思決定を記録」のあと、誰に・何を・いつまで を伝える → `意思決定ログ/` に追記 |
| **コミットメントのリマインダー** | 「リマインダー作成」→ 前回決めたやことをまとめた文面を出力（送信先は `設定/slack.md`・テスト用 C0ADM5X1XJA） |
| **週次・月次の数値要約** | 数値を貼って「週次要約」または「月次要約」→ 短いレポートに要約（`週次レポート/`・`月次レポート/` に保存想定） |
| **ナレッジをNotion/Spreadsheetへ** | 「ナレッジをエクスポート」→ CSVまたは表形式で出力 |

**設定の変更（後ほど修正）**：Slack送信先・次回MTG日などは **`設定/`** フォルダのファイルを編集。一覧は `設定/README.md`。

## 事前準備（初回のみ）

### 1. 拡張機能のインストール

**一覧・選び方** → **★MTGフロー・ルール.md** の「このワークフロー実行に最適な拡張機能」を参照。

Cursor でこのフォルダを開いた状態で：

1. **Cmd + Shift + X** で拡張機能パネルを開く  
2. 検索欄で **「Whisper」** または **「Speech」**（録音・文字起こし）、**「Slack」**（議事録・Todo の送信）で検索  
3. 目的に合う拡張を **インストール**。ワークスペースの「推奨」に表示されていればそこから一括でも可  

| 用途 | 検索キーワード | 備考 |
|------|----------------|------|
| 録音・文字起こし | Whisper / Speech | 日本語は Whisper 系か VS Code Speech。詳細は ★ 参照 |
| 議事録・Todo を Slack に送る | Slack / vscode2slack | VSCode 2 Slack など。トークン設定が必要 |

### 2. Speech to Text with Whisper を使う場合

- **FFmpeg** をインストール（音声処理用）  
  - macOS: `brew install ffmpeg`  
- **OpenAI API キー** を設定  
  - Cursor の設定で「Whisper」や「OpenAI」を検索し、API キーを入力  

### 3. 日本語にする（VS Code Speech の場合）

設定（**Cmd + ,**）で次を検索して設定：

- `accessibility.voice.speechLanguage` → **ja-JP**

## 本番フロー（会議当日）

### Step 1：録音 → 文字起こし

1. 会議中に **PCのマイクで録音**（拡張の「録音開始」を使用）  
2. 終了したら **録音停止** → 拡張が文字起こしを実行  

**Speech to Text with Whisper の場合：**

- **Cmd + Shift + N** … 録音して **そのままAIチャットに送信**（推奨）  
- または **Cmd + Shift + M** … 録音してカーソル位置にテキスト挿入 → 必要ならそのテキストをチャットにコピペ  

**VS Code Speech の場合：**

- **Cmd + Alt + V**（エディタへ）または **Cmd + I**（チャットへ）で音声入力  
- 書き起こし結果をコピーし、Cursor の AI チャットに貼る  

### Step 2：① 議事録まとめ ＋ 次回までにやること（Todo）

チャットに次のように送る（文字起こしテキストをその前か後に貼る）：

```
以下は幹部MTGの音声文字起こしです。
①議事録にまとめ、次回までにやるべきことをメンバー別に整理し、Slack用のTodo文面を作成してください。
```

→ 出力された「議事録」と「Slack用Todo」をコピーし、Slackで各メンバーに送信。

### Step 3：② ナレッジ化

同じチャットまたは続けて：

```
上記の議事録（または今回の議事録要約）を、社内ナレッジとして保存する形式で整理してください。
日付・会議名・決定事項・アクション・参照用キーワードを含めてください。
```

→ 出力をこのフォルダの「ナレッジ」用ファイル（例：`ナレッジ/YYYY-MM-DD_幹部MTG.md`）に保存すると、後から検索しやすくなります。
