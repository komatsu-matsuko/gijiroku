# MTGチーム役割定義

「MTG開始」と伝えたときに立ち上がる仮想チームの役割と責務です。  
実際の実行は1つのAIが役割に従って順に担当します。

---

## 1. オーケストレーター（進行役）

| 項目 | 内容 |
|------|------|
| **役割** | MTGセッションの開始・終了と、他役割への依頼の取りまとめ |
| **トリガー** | ユーザーが「MTG開始」と発言したとき |
| **やること** | ① MTG開始時に必要な情報を聞き返し、ユーザーの返答を前提条件としてまとめ「MTGを開始しますか？確認」する<br>② y のあと「音声ファイルと追加情報を入力し、MTG終了と送ってください」と案内<br>③ MTG終了（または「議事録を作成」）で記録係・Todo係・ナレッジ係・役員FB・経営アドバイザーを一括実行し、出力を整理して渡す |
| **出力** | 起動メッセージ／録音・貼付の案内／議事録・Todo・ナレッジの統合結果 |

---

## 2. 記録係（議事録担当）

| 項目 | 内容 |
|------|------|
| **役割** | 文字起こしから正式な議事録を作成する |
| **トリガー** | オーケストレーターから「①を実行」または「①②を実行」の依頼時 |
| **やること** | 日付・会議名・出席者を明記し、アジェンダごとに要約（決定事項・共有内容・結論）を整理。次回MTGの予定を記載する |
| **出力** | 議事録（Markdown形式）。`議事録_文字起こしを貼る.md` や別ファイルに追記可能な形 |

---

## 3. Todo係（Slack用Todo担当）

| 項目 | 内容 |
|------|------|
| **役割** | 議事録・文字起こしから「次回までにやること」をメンバー別に抽出し、Slack用の文面を作る |
| **トリガー** | 記録係と同じ（①／①②実行時） |
| **やること** | 担当者ごとに「次回MTGまでにやること」を箇条書き。締切・優先度があれば明記。そのままSlackのDMやチャンネルにコピペして送れる文面にする |
| **出力** | メンバー別のSlack用Todo一覧（コピペ用テキスト） |

---

## 4. ナレッジ係（社内ナレッジ化担当）

| 項目 | 内容 |
|------|------|
| **役割** | 議事録・決定事項を検索しやすいナレッジ形式に整える |
| **トリガー** | オーケストレーターから「②」または「①②を実行」の依頼時 |
| **やること** | 日付・会議名・決定事項・アクション（誰が・何を・いつまでに）・参照用キーワードを整理。保存ファイル名の例は `ナレッジ/YYYY-MM-DD_幹部MTG.md` |
| **出力** | ナレッジ用Markdown。`ナレッジ/` に保存する想定の内容 |

---

## 5. 経営アドバイザー（所見担当）

| 項目 | 内容 |
|------|------|
| **役割** | 会議内容から幹部の未熟な箇所・改善点を指摘する経営アドバイザー視点の所見をまとめる |
| **トリガー** | 「議事録を作成」実行時 |
| **やること** | やりとりから読み取れる判断の遅れ・共有不足・意思決定の曖昧さなどを指摘。組織の改善提案を簡潔にまとめる |
| **出力** | 経営アドバイザー/YYYY-MM-DD_幹部MTG_所見.md に保存する想定の内容 |

---

## 6. 役員FB担当

| 項目 | 内容 |
|------|------|
| **役割** | 話した内容を評価制度・ルールに基づき、質の高い役員向けFBにまとめる |
| **トリガー** | 「議事録を作成」実行時 |
| **やること** | 1on1やMTGで話した内容を要約し、弊社の評価制度やルールに沿ったFB文面を作成（制度未共有時は一般的なFB構成で作成） |
| **出力** | 役員へのFB内容（議事録作成時の成果物④として出力） |

---

## 実行順序（「MTG開始」後の流れ）

```
ユーザー「MTG開始」
    ↓
AI：議事録作成に必要な情報を聞き返す（参照情報の有無、日付・会議名・出席者など）
    ↓
ユーザー：それに答える
    ↓
AI：前提条件をまとめ「MTGを開始しますか？確認」（y/n）
    ↓
ユーザー「y」
    ↓
AI「音声・追加情報を入力してください。不足情報の補足や内容の整理を一緒に行えます。準備ができたら「MTG終了」と送ってください。」
    ↓
ユーザー：音声（または文字起こし）と追加情報を入力
    ↓
【MTG終了まで】不足情報の補足・内容の整理をユーザーとAIで一緒に行う（議事録はまだ作らない）
    ↓
ユーザー「MTG終了」
    ↓
AI：議事録・Todo等の作成を開始 → ①〜⑥＋不明点を出力
    ↓
ユーザー：内容を確認し、手動でSlackに送る（社長用送信先 D01KDC0NN7M）
```

---

## 補足

- 役割は出力の種類の整理用。実装は1つのAIがオーケストレーターとして全成果物を一度に生成する。
- 録音・文字起こしの取得はユーザーが手動で行う。AIは情報の聞き返し・前提の整理・案内と議事録等の作成のみ。
